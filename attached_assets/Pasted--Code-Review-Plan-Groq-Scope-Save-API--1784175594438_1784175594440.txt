## نقاط الفحص

### 1. مصدر قيمة المفتاح

ابحث عن:

```tsx
const [apiKey,setApiKey]
```

أو:

```tsx
useState("")
```

تحقق:

* هل القيمة محلية فقط؟
* هل يتم تحميلها من API عند فتح الصفحة؟

المشكلة المحتملة:

```tsx
const [groqKey,setGroqKey] = useState("")
```

بدون:

```tsx
useEffect(fetchProviders)
```

---

## 2. زر Save

ابحث عن:

```tsx
onClick={save}
```

أو:

```tsx
mutation.mutate()
```

تحقق من:

### Request Payload

مثال صحيح:

```json
{
 provider:"groq",
 apiKey:"gsk_xxx"
}
```

تحقق من وجود:

```
userId
projectId
workspaceId
```

حسب تصميم المشروع.

---

## 3. بعد نجاح الحفظ

ابحث عن:

```tsx
onSuccess
```

تحقق:

هل يحدث:

```tsx
invalidateQueries()
```

أم فقط:

```tsx
toast.success()
```

المشكلة المحتملة:

الواجهة تقول:

```
Saved successfully
```

لكن السيرفر لم يحفظ.

# علامات الخطأ التي سأبحث عنها أثناء المراجعة

| العلامة                     | معناها المحتمل                      |
| --------------------------- | ----------------------------------- |
| Save يعطي نجاح لكن DB فارغة | API لا يحفظ                         |
| DB تحتوي سجل لكن GET فارغ   | Query/filter مشكلة                  |
| GET يعمل قبل refresh فقط    | state محلي                          |
| GET يرجع null               | decrypt أو mapping                  |
| chat يفشل فقط               | requireGroqApiKey لا يجد credential |
| أكثر من سجل Groq            | لا يوجد upsert                      |

## نتيجة التحقيق المطلوبة في النهاية

سيكون التقرير النهائي بهذا الشكل:

```
ROOT CAUSE:

File:
xxxxx.ts

Function:
saveProvider()

Problem:
API key stored only in React state because DB insert never executes.

Fix:
Replace X with Y.

Verification:
1. Save key
2. Refresh
3. GET /providers returns configured=true
4. Chat request succeeds
```

هذه الخطة هي مسار مراجعة فعلي يمكن تنفيذه على المستودع خطوة بخطوة بدون الرجوع للتخمين.
